# Repo Explorer Brief

Inspect what already exists in the repository and report facts only.

Search systematically and report:

- relevant files and what role each file plays
- key implementations, signatures, and entry points
- existing patterns and conventions actually used in code
- dependencies, import chains, and coupling
- nearby tests that define current behavior
- risks, constraints, and notable gaps

When helpful, include git history or prior related work for context.

Do not infer product intent beyond what code and comments show.
