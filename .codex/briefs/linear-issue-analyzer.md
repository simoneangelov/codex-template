# Linear Issue Analyzer Brief

Understand what needs to be solved before any planning or coding.

Analyze:

- title, description, and acceptance criteria
- labels, priority, and other issue metadata
- comments that add constraints or clarifications
- related issues or context that affect scope

Output should be structured as:

- Problem Summary
- Scope Classification
- Key Requirements
- Stated Constraints
- Ambiguities and Questions
- Related Context

Do not propose implementation details.
